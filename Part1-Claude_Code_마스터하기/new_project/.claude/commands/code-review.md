다음 순서로 코드 리뷰를 수행하세요:

## 1. 리뷰 대상 파악
- 인자가 있으면 해당 파일을 리뷰 대상으로 삼으세요: $ARGUMENTS
- 인자가 없으면 `git diff HEAD` 또는 `git diff --staged`로 현재 변경사항을 확인하세요.
- 변경사항이 없으면 프로젝트의 핵심 파일을 직접 읽어 리뷰하세요.

## 2. 리뷰 체크리스트

### 🐛 버그 & 정확성
- 논리 오류, 엣지 케이스 누락, 잘못된 조건문
- 비동기 처리 오류 (race condition, unhandled promise 등)
- 타입 불일치, null/undefined 참조 위험

### 🔒 보안
- XSS, SQL 인젝션, 민감 정보 노출 등 OWASP Top 10
- 사용자 입력 검증 누락
- 안전하지 않은 외부 데이터 처리

### ⚡ 성능
- 불필요한 재렌더링 또는 반복 연산
- 메모리 누수 가능성
- 최적화 가능한 반복문이나 DOM 조작

### 🧹 코드 품질
- 가독성 및 명명 규칙
- 중복 코드
- 불필요한 복잡성

## 3. 결과 형식

각 이슈를 아래 형식으로 보고하세요:

```
[심각도: 🔴 치명 / 🟡 경고 / 🔵 제안]
파일: <파일명>:<라인번호>
문제: <한 줄 요약>
현재 코드: <문제 코드>
수정 제안: <개선 코드 또는 방향>
```

## 4. 요약
- 발견된 이슈 수 (치명/경고/제안)
- 가장 우선 수정해야 할 항목 1~3개
- 전반적인 코드 품질 평가 (1~5점)
