# 바이브 코딩 90분 실습 — 전월세 비교 계산기

이 실습은 Claude Code를 활용한 바이브 코딩(Vibe Coding) 세션으로 진행했습니다.

> **실습 결과물은 별도의 git 저장소에서 작업했습니다.**
> - 로컬 경로: `../../../jeonse-calculator`
> - 원격 저장소: https://github.com/Chbee/jeonse-calculator

## 무엇을 만들었나

**전월세 비교 계산기** — 자기자본과 대출금리를 기준으로 여러 매물의 전세·월세를 비교해 가장 유리한 선택을 찾아주는 단일 HTML 웹앱입니다.

### 주요 기능
- **전역 입력**: 자기자본, 대출금리, 대안수익률(r_alt) 기본값 설정
- **매물 카드**: 매물을 여러 개 추가하고 전세 / 월세 / 전세↔월세 비교 유형 선택
- **최종 선택 추천**: 입력값을 바탕으로 가장 유리한 시나리오 자동 추천
- **매물 간 비교 표** 및 **월 고정 지출 비교 차트**(Chart.js)
- **JSON 내보내기/가져오기**: 파일 업로드·드래그앤드롭 지원, Notion 매물 데이터 import
- **결과 해석 가이드**: 대출이자, 전월세 전환율 등 숫자를 읽는 법 안내

### 핵심 계산 로직
- 전세 대출액 = `max(0, 전세보증금 − 자기자본)`, 이자가 월 고정 지출
- 전월세 전환율 = `(월세 × 12) ÷ (전세보증금 − 월세보증금) × 100`
- 전환율 > 대출금리 → 전세가 유리 / 전환율 < 대출금리 → 월세가 유리

## 기술 스택
- 단일 `index.html` (바닐라 JS + CSS)
- [Chart.js](https://www.chartjs.org/) 4.4.0 (CDN)

## 실습 진행 방식
별도 저장소(`jeonse-calculator`)에서 Claude Code와 함께 점진적으로 기능을 추가하며 커밋했습니다:
1. `Initial commit: 전세 계산기`
2. `JSON 가져오기에 파일 업로드·드래그앤드롭 추가 및 Notion 매물 데이터 반영`
3. `전역 입력값(자기자본·대출금리·r_alt) 기본값 설정`
